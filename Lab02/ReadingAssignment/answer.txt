When should accessor be used?
Answer: According to Allen Holub, accessor methods are not recommended to be used unless absolutely necessary situations. This is because these methods expose information about how a class is implemented and as a consequence make the code more difficult to maintain.

If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?
Answer: Yes, Java allows us to create multiple constructor methods with different parameters with a feature known as constructor overloading. This means we can have more than one constructor in a class, as long as each constructor has a unique parameter list.
